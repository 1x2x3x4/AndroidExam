## 三、实验内容

本实验实际完成的商品包括：

1. 百岁山矿泉水
2. 面包
3. 有机纯牛奶

## 五、系统功能设计

### 5.1 界面整体设计

主界面采用纵向 `LinearLayout` 作为根布局，上部为标题和搜索框，中部为 `ScrollView + LinearLayout` 的商品列表区域，下部为购物车总价显示栏。整体界面结构清晰，便于课程实验展示和功能演示。

界面主要分为三个区域：

1. 顶部区域：显示标题“超市商品列表”和搜索输入框。
2. 中间区域：动态生成商品卡片列表，每张卡片展示图片、名称、价格、数量控制和加入购物车按钮。
3. 底部区域：显示购物车总价，并根据用户操作实时更新。

### 5.2 商品展示设计

每个商品项包含以下内容：

- 左侧：商品图片。
- 右侧上方：商品名称。
- 右侧中部：商品价格，使用红色高亮显示。
- 右侧下方：数量减按钮、数量显示、数量加按钮。
- 最下方：加入购物车按钮。

为提升界面效果，商品项使用圆角浅色卡片背景；搜索框、按钮、总价栏分别配置了对应的圆角背景资源，使界面更加统一、美观。

### 5.3 功能逻辑设计

1. 商品数据初始化：在 `MainActivity` 中定义 `Product` 内部类，并初始化五个商品对象。
2. 动态渲染列表：根据商品集合动态生成每一项卡片视图并添加到列表容器中。
3. 数量增减控制：通过 “-” 和 “+” 按钮修改当前商品数量，数量最小值限制为 0。
4. 加入购物车：点击按钮后，若数量为 0，则提示“请选择商品数量”；否则按“单价 × 数量”累加到总价中。
5. 总价刷新：每次成功加入购物车后，调用方法刷新底部总价文本。
6. 搜索筛选：通过监听搜索框内容变化，按商品名称进行匹配，只显示符合条件的商品项。

## 六、核心实现过程

### 6.1 布局文件实现

在 `activity_main.xml` 中，使用 `LinearLayout` 构建整体页面结构，并在中间部分通过 `ScrollView` 包裹商品列表容器。这样既满足实验文档对 `ScrollView + LinearLayout` 的要求，也保证商品较多时页面可以上下滚动。

布局中的关键控件包括：

- `TextView`：页面标题、总价显示。
- `EditText`：商品搜索输入框。
- `ScrollView`：页面可滚动区域。
- `LinearLayout`：商品列表动态挂载容器。

### 6.2 颜色与字符串资源配置

在 `colors.xml` 中定义了页面背景色、卡片边框色、主色调、价格红色、提示色等资源，避免在布局和代码中直接写颜色值，提高了可维护性。

在 `strings.xml` 中统一定义标题、按钮文案、价格格式、总价格式、搜索提示和商品名称等字符串资源，便于后续修改和国际化扩展。

### 6.3 商品数据模型设计

在 `MainActivity` 中定义了 `Product` 静态内部类，包含以下字段：

- `nameResId`：商品名称字符串资源 ID
- `price`：商品单价
- `imageResId`：商品图片资源 ID
- `quantity`：当前选择数量

这种数据结构简单明确，适合当前实验项目，也便于通过循环统一渲染界面。

### 6.4 动态创建商品卡片

项目没有在 XML 中写死每个商品项，而是在 Java 代码中通过 `createProductCard(Product product)` 方法动态创建商品卡片。其主要流程如下：

1. 创建商品卡片根布局并设置圆角背景。
2. 创建 `ImageView` 显示商品图片。
3. 创建信息区域，显示商品名称与价格。
4. 创建数量控制区域，绑定加减按钮事件。
5. 创建“加入购物车”按钮并绑定点击事件。
6. 将完整卡片添加到商品列表容器中。

这种实现方式减少了重复 XML，便于统一管理样式与逻辑，也便于后续扩展商品数量。

### 6.5 搜索功能实现

搜索功能通过给 `EditText` 添加 `TextWatcher` 监听器实现。当输入框内容变化时，程序会调用 `renderProducts()` 方法，重新根据关键字过滤商品名称并刷新显示列表。

当搜索结果为空时，系统会显示“没有找到相关商品”的文本提示，提升交互完整性。

### 6.6 加入购物车与总价统计

加入购物车逻辑在按钮点击事件中实现：

1. 首先判断当前商品数量是否为 0。
2. 若为 0，则弹出 `Toast` 提示用户先选择数量。
3. 若数量大于 0，则将 `商品数量 × 商品价格` 累加到 `cartTotalPrice`。
4. 调用 `updateTotalPrice()` 方法刷新页面底部总价显示。
5. 加入购物车成功后，将当前商品数量重置为 0，避免重复累计造成误操作。

该设计符合实验场景需求，逻辑简洁，用户操作反馈明确。

## 九、实验中遇到的问题及解决方法

### 9.1 问题一：如何在同一页面中展示多个商品项

最初如果直接在 XML 中手工编写多个商品块，会导致布局代码重复较多，后续修改也不方便。为解决这一问题，项目采用 Java 动态创建商品卡片的方式，通过循环渲染商品列表，提高了代码复用性。

### 9.2 问题二：如何实现搜索后界面实时刷新

搜索功能的关键在于监听输入内容变化。项目通过 `TextWatcher` 监听搜索框文本，并在文本变化后重新过滤商品数据并刷新布局，从而实现搜索结果的即时反馈。

### 9.3 问题三：如何避免用户在数量为 0 时加入购物车

若不做校验，用户点击按钮后会出现逻辑不明确的问题。项目中在加入购物车前先判断数量是否为 0，若为 0 则通过 `Toast` 给出提示信息，从而保证操作逻辑合理。

### 9.4 问题四：如何统一页面视觉风格

如果直接使用系统默认按钮和背景，界面会比较生硬。项目通过新增多个 `drawable` 圆角背景文件，并结合颜色资源统一设置卡片、搜索框、按钮和总价栏样式，使页面观感更完整。
